    # 自动化部署

    为了减少在这部分的学习成本，我们使用一个极简的自动化工具

    不像晦涩难懂庞大完整的 CI/CD，这个小工具是基于 ssh 的纯 javascript 安全复制工具，配置文件清爽易读

    首先在项目根目录安装

    yarn add scp2 ora@5.1.0 chalk@4.1.2 -D
或
npm i scp2 ora@5.1.0 chalk@4.1.2 -D

    ora 和 chalk 是为了控制台显示看起来好看，而且在 win7 下不指定版本号安装的最高版本执行会报错，不喜欢的可以不装，releases 文件夹下的配置文件把相关引用和执行删除即可

    1. 根目录建立目录和文件/releases/index.js 粘贴代码
    const scpClient = require('scp2') // 服务器连接模块
const ora = require('ora') // 发布过程中打印加载动画
const chalk = require('chalk') // node控制台样式模块
const spinner = ora('正在发布到linux服务器...')

spinner.start()
scpClient.scp(
    'dist/', // 打包后文件存放的目录
    {
        host: '', // 服务器ip地址
        port: 22,
        username: '', // 服务器用户名
        password: '', // 服务器用户密码
        path: '', // 服务器nginx代码存放地址
    },
    (err) => {
        // 发布代码后的回调函数
        spinner.stop()
        if (err) {
            console.log(chalk.red('发布失败.\n'))
        } else {
            console.log(chalk.blue('Success! 成功发布到linux服务器! \n'))
        }
    }
)

    • path（上传项目到服务器的哪个目录下）
    1. 找到根目录的 package.json 的 scripts 中最后增加一段脚本 "release" 并保存
    "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "cross-env NODE_ENV=production && webpack --mode=production --config webpack.prod.config.js",
    "serve": "cross-env NODE_ENV=development && webpack serve --mode=development --config webpack.dev.config.js",
    "release": "cross-env NODE_ENV=production && webpack --mode=production --config webpack.prod.config.js && node ./release"
}

    代码修改一下

    const scpClient = require('scp2') // 服务器连接模块
const ora = require('ora') // 发布过程中打印加载动画
const chalk = require('chalk') // node控制台样式模块
const readline = require('readline')
const spinner = ora('正在发布到linux服务器...')

const deployFile = (username, password) => {
    spinner.start()
    scpClient.scp(
        'dist/', // 打包后文件存放的目录
        {
            host: '', // 服务器ip地址
            port: 22,
            username: username, // 服务器用户名
            password: password, // 服务器用户密码
            path: '', // 服务器nginx代码存放地址
        },
        (err) => {
            // 发布代码后的回调函数
            spinner.stop()
            if (err) {
                console.log(chalk.red('发布失败.\n'))
            } else {
                console.log(chalk.blue('Success! 成功发布到linux服务器! \n'))
            }
            rl.close()
        }
    )
}

//创建readline实例
const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
})
const questions = ['需输入服务器用户名: ', '需输入服务器密码: ']
const linelimit = 2
let inputArr = []
let index = 0

// 依次执行命令行交互语句
const runQueLoop = () => {
    if (index === linelimit) {
        deployFile(inputArr[0], inputArr[1])
        return
    }
    rl.question(questions[index], (as) => {
        inputArr[index] = as
        index++
        runQueLoop()
    })
}

rl.on('close', () => {
    process.exit(0)
})
runQueLoop()

    关于服务器备份

    webpack 每次执行 release 时打包后的文件名会根据业务代码的修改情况动态的生成 hash，导致备份不清晰，万一需要临时回退版本很难搞，上面的操作只把文件复制过去，遇到同名文件才会覆盖。

    我们还需要一个每次部署都把服务器上的目录备份到一个文件夹内，删除掉目录的业务文件，把新的文件替换上去
